## Что такое Искра — в одном абзаце

P2P мессенджер для Android без серверов. Сообщения шифруются на устройстве отправителя, расшифровываются на устройстве получателя. Между ними — mesh-сеть из таких же телефонов, каждый из которых носит в "трюме" чужие зашифрованные сообщения и передаёт их дальше при контакте с другими телефонами (Wi-Fi, Bluetooth). Как FidoNet, только на телефонах и с современной криптографией.

## Аналогия для понимания

Почтовые корабли XVIII века. Письмо запечатано (шифрование). Отдано капитану (соседний телефон). Капитан не знает что внутри. Везёт в следующий порт (Wi-Fi сеть). Передаёт другому капитану. Рано или поздно доходит до адресата. Нет центральной почты. Нет одного маршрута. Потопишь один корабль — письмо уже на десяти других.

## Контекст по криптографии — ВАЖНО

Мы долго обсуждали выбор крипто. Вот итог:

Российская военная рация "Азарт" (Р-187-П1), позиционируемая как защищённая связь, оказалась с "шифрованием" на 6-значном ключе (миллион комбинаций, ломается за секунды). Тайваньские комплектующие, американский FPGA Xilinx Spartan, китайская сборка. Украина перехватывает и дешифрует в реальном времени. Трофейные экземпляры продаются на аукционах.

Вывод: нельзя доверять ничему проприетарному. Нельзя доверять "отечественным разработкам." Нельзя доверять чёрным ящикам.

**Решение: NaCl/libsodium.** XSalsa20-Poly1305 + X25519 + Ed25519. Создано Даниэлем Бернштейном специально как ответ на недоверие к NIST/АНБ. Открытый код. Открытая математика. Аудировано сотнями независимых криптографов. Сноуден доверяет.

**Плюс слой обфускации поверх.** Не вместо — поверх. XOR с HMAC(shared_secret, timestamp || counter). Чтобы выходной поток не был похож на стандартный libsodium output. DPI или реверс-инженер, знающий libsodium, не узнает его с ходу. А под обфускацией — настоящая криптография, которая не ломается.

Два замка на двери. Внешний — нестандартный, сбивает с толку. Внутренний — танковая броня.

## Что мы строим в этой итерации (альфа v0.1)

Жёстокий минимум. Proof of concept. Два телефона обмениваются зашифрованными текстовыми сообщениями без сервера. Всё остальное — в следующих версиях через ползучий апдейт.

### Есть в альфе:

1. **Генерация ключей** — Ed25519 keypair. Публичный ключ = ID пользователя (base58, 20 символов). Мнемоника: 24 русских слова.

2. **Шифрование** — XSalsa20-Poly1305 (E2E). Обфускация поверх. Подпись Ed25519.

3. **LAN Discovery** — UDP multicast (239.42.42.42:4242), beacon раз в 60 сек. Автонаходка в одной Wi-Fi сети.

4. **Wi-Fi Direct** — если возможно (Android API). Поиск раз в 60 сек, предложение соединения.

5. **Транспорт** — Reliable UDP (KCP) для связи между нодами.

6. **Мешок почты (store-and-forward)** — каждый телефон хранит чужие зашифрованные сообщения и передаёт при контакте с другими нодами. Bloom filter для дедупликации. Пока без автоудаления (но TTL-поле в структуре сообщения заложено на будущее).

7. **Минимальный UI** — Telegram-like. Список контактов, окно чата, поле ввода. Без красивостей, но юзабельный.

8. **Импорт контактов** — из JSON-файла (сгенерированного утилитой "Искра-Мост").

9. **Механизм обновления** — чтобы следующие версии можно было доставить через mesh.

## Принципы разработки

1. **Один APK.** Пользователь получает один файл. Устанавливает. Всё работает. Без зависимостей, без серверов, без интернета (если есть хотя бы один другой телефон с Искрой рядом).

2. **Go ядро + Android WebView.** Ядро на Go (криптография, сеть, хранение). Компилируется через gomobile в .aar. Минимальная Kotlin-обёртка: запускает Go-ядро как сервис, показывает WebView на localhost. Интерфейс — HTML/CSS/JS, встроенный в Go через embed.

3. **Каждый модуль тестируем.** `go test ./...` должен проходить на каждом шаге. Не пиши следующий модуль пока текущий не покрыт тестами.

4. **Никаких логов по умолчанию.** Демон не пишет на диск ничего кроме зашифрованного хранилища. Логи — только при --debug.

5. **Улучшай.** Видишь способ сделать лучше, быстрее, надёжнее — делай. Потом напиши комментарий в коде: "// IMPROVEMENT: [что и почему]". Мы ревьюим и учимся.
