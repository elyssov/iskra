## Build 12 — PIN + Паника + Шифрование (Приоритет 2 роадмапа)

### PIN-код при запуске
- 4-6 цифр, тёмный экран с цифровой клавиатурой (стиль банковского приложения)
- Первый запуск → установка + подтверждение
- Последующие → ввод для разблокировки
- **5 неверных попыток = автоматическое уничтожение данных**
- Анимации: shake при ошибке, зелёная подсветка при успехе
- Поддержка физической клавиатуры, автосабмит при 4+ цифрах

### Режим паники (Panic Mode)
- Зажать логотип «Искра» на 3 секунды → поле ввода кода
- Код "159" → мгновенное уничтожение
- **Secure wipe:** 3 прохода `crypto/rand` по каждому файлу → `Sync()` → `Remove()`
- Перезаписанные данные невозможно восстановить forensic-инструментами

### Plausible Deniability (генерация шума)
После паники или 5 неверных PIN генерируются:
- **4 фейковых контакта:** Маша, Дима, Алёна, Серёга
- **~94 сообщения** small talk: погода, Дюна, рецепт плова, любительский футбол
- **150-350 фейковых "зашифрованных" сообщений** в трюме (рандомные байты)
- Приложение перезагружается и выглядит как обычный мессенджер со скучной перепиской
- Товарищ майор видит Барсика на столе, не координацию митинга

### Шифрование локального хранилища
- `storage_key = HMAC-SHA256(seed, "iskra-storage-v1:" + PIN)`
- Контакты, сообщения, группы: XSalsa20-Poly1305
- Бесшовная миграция: старые plaintext файлы читаются, при сохранении шифруются
- На диске — только зашифрованные блобы

### Standalone демо
- `cmd/panic-demo/` — запускает полный UI Искры с decoy-данными
- Безопасно: работает во временной директории, не трогает реальные данные
- Можно показать заказчикам как выглядит Искра "глазами майора"

### Тесты
7 новых тестов: PIN set/verify, attempts, vault encrypt/decrypt, key derivation, file roundtrip, wipe, decoy generation.

## Build 12.1 — UDP Direct Transport (Приоритет 3 роадмапа)

### Обфускированный UDP транспорт
- **Протокол:** `[nonce(8)] + XOR(data, HMAC_keystream)`
- Каждый пакет с уникальным nonce — DPI видит случайные байты разной длины
- Без хэндшейка, без сессий, без TCP-паттернов
- Авто-регистрация на relay, keepalive 25с

### UDP Relay сервер
- `cmd/udp-relay/` — standalone сервер, один UDP-порт
- Де-обфускация → маршрутизация по recipient_id → ре-обфускация → пересылка
- Очистка мёртвых клиентов через 5 минут
- Готов к деплою на любой VPS

### Интеграция
- Флаг `--udp-relay host:port`
- Работает параллельно с WebSocket relay
- 4 новых теста (roundtrip, uniqueness, randomness, short packet)

## Build 13 — Mesh Wave Protocol (Приоритет 4 роадмапа)

### Wave Controller (`mesh/wave.go`)
Decentralized mesh для работы БЕЗ интернета (сценарий митинга):

- **State machine:** CHECK_WIFI → INFRA → DIRECT → SLEEP → loop
- **Авто-переключение:** есть Wi-Fi → multicast discovery; нет Wi-Fi → Wi-Fi Direct P2P
- **Cooldown per peer** (10 мин): предотвращает повторный sync с тем же устройством
- **Cooldown RESET при новых сообщениях:** запускает новую волну распространения
- Случайный выбор peer → минимизация конфликтов

### Bloom filter dedup fix
- **БЫЛО:** transport отправлял ВСЕ сообщения из трюма каждому peer'у
- **СТАЛО:** проверяет bloom filter получателя, отправляет только НОВЫЕ
- 1000 телефонов × 100 сообщений: было ~100M передач, стало ~100K

### Wi-Fi Direct (`WifiDirectManager.kt`)
- Discovery → Connect → Sync → Disconnect → Next peer
- Не нужен роутер, работает прямо между телефонами
- Kotlin BroadcastReceiver для P2P состояний

### Волновой эффект
Сообщение от X за ~60 сек доходит до 1000 телефонов через экспоненциальное распространение.

## Build 14.5 — Hotfixes

### Чат не переключался (КРИТИЧНЫЙ)
- **Причина:** `#welcome-screen` был дочерним элементом `#messages`. При первом `renderMessages()` → `innerHTML = ...` → welcome-screen уничтожался → JS error → все последующие openChat() падали молча.
- **Фикс:** welcome-screen вынесен в sibling-элемент.

### Горизонтальные бабблы
- **Причина:** два CSS-блока `#messages`, один ставил `display: flex` (default direction = row).
- **Фикс:** слито в один блок, `display: block`.

### Клавиатура закрывает ввод (Android)
- `windowSoftInputMode=adjustResize` в AndroidManifest
- `visualViewport` API fallback: JS ловит resize, добавляет padding + scrollIntoView

### APK подпись (FOTA несовместимость)
- **Причина:** GitHub Actions debug keystore разный на каждом runner'е → разная подпись → Android блочит обновление.
- **Фикс:** постоянный keystore (`iskra-release.jks`), хранится в GitHub Secrets, CI подписывает release APK.

### FOTA спам
- **Причина:** предлагала обновление при каждом запуске, даже если уже обновился.
- **Фикс:** localStorage хранит dismissed version, не показывает повторно.
