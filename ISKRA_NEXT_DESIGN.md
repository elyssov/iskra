## ФИЛОСОФИЯ

Искра — не приложение. Искра — окно в мир для людей, у которых мир пытаются отнять.

Текущий UI — MVP. Одна вкладка, всё в куче. Контакты, чаты, каналы — мешанина. Для альфы было нормально. Для людей, которые расклеивают QR по городу — нет.

Новый UI должен быть:
- **Интуитивным** — "для учительницы из Воронежа"
- **Быстрым** — бюджетный Honor с Snapdragon 680 не должен тормозить
- **Красивым** — люди должны ХОТЕТЬ пользоваться
- **Живучим** — работать без relay, без интернета, без всего

### Протокол писем

Новый ContentType: `0x20 = ContentLetter`

**TTL по типу контента:**
| Тип | ContentType | TTL (kill switch) | Смысл |
|-----|-------------|-------------------|-------|
| Чат | 0x01 Text | 15 дней | Мгновенное, не дошло — неактуально |
| Письмо | 0x20 Letter | 60 дней | Надёжная доставка, может ждать |
| Группа | 0x03 GroupText | 15 дней | Как чат |
| Файл | 0x05 FileChunk | НЕ хранится в hold | Только direct delivery |
| Канал | 0x10 ChannelPost | 30 дней | Broadcast, средний приоритет |
| FOTA | 0x30 FOTA | 7 дней | Обновление ПО |

Payload (JSON внутри шифрованного контейнера):
```json
{
  "subject": "Тема",
  "body": "Текст письма...",
  "reply_to": "original_letter_id_or_null",
  "attachments": [
    {"name": "file.pdf", "size": 12345, "chunk_ids": ["abc", "def"]}
  ]
}
```

## QR-КОДЫ

### Генерация
- JS-библиотека `qrcode.js` (~5КБ, MIT, без зависимостей)
- Генерируется на клиенте из `iskra://` ссылки
- Для контактов: `iskra://{edPub}/{x25519Pub}/{displayName}`
- Для каналов: `iskra://channel/{channelID}/{ownerPub}`
- Для писем: `iskra://letter/{recipientPub}` (откроет редактор с заполненным "Кому")

### Отображение
- В модалке "Моя визитка" — QR + ссылка + кнопка "Копировать"
- В контекстном меню контакта — "Показать QR"
- Под QR — короткий адрес `iskra:lara`
- Под ним — полная ссылка мелким шрифтом

### Сканирование
- **Android:** Intent ACTION_VIEW с камерой → распознаёт QR → передаёт iskra:// в deep link handler
- **Десктоп:** кнопка "Вставить ссылку" (камеры нет, только clipboard)

### Плакаты
Формат для печати: QR + текст "Скачай Искру → Сканируй → Общайся свободно"

## КОРОТКИЕ АДРЕСА

### Формат
```
iskra:lara
iskra:kot
iskra:4417
iskra:revolution
```

### Механика
1. Пользователь выбирает себе tag (3-20 символов, латиница/цифры)
2. POST `/api/tag/register` → relay сохраняет `tag → {edPub, x25519Pub, displayName}`
3. При добавлении контакта: ввод `iskra:lara` → GET `/api/tag/resolve/lara` → полные ключи
4. Tag уникален (first come first served)
5. Tag привязан к pubkey, не к сессии
6. TTL: пока пользователь активен (30 дней без появления → освобождается)

### Резервация
Зарезервированные теги:
- `master` → Мастер (DEV)
- `lara` → Лара (AI)
- Имена из списка aliases relay (Ильич, Че, Спартак...) → блокированы

### На relay
```go
// Новые endpoints
mux.HandleFunc("/api/tag/register", r.handleTagRegister)  // POST {tag, edPub, x25519Pub}
mux.HandleFunc("/api/tag/resolve/", r.handleTagResolve)    // GET /api/tag/resolve/{tag}
```

### Без relay (mesh-only)
Короткие адреса не работают. Только полные `iskra://` ссылки и QR. Это ожидаемо — без DNS нет доменных имён.

## P2P FOTA (ОБНОВЛЕНИЕ ЧЕРЕЗ MESH)

### Проблема
Relay умрёт при чебурнете. GitHub недоступен. Как обновлять?

### Решение
Нода с новой версией — раздаёт APK другим нодам при mesh sync.

### Протокол
1. При sync: обмен build numbers вместе с bloom (добавить в HELLO)
2. Если `peer.build > my.build` → запросить обновление
3. Peer отправляет APK чанками (ContentType `0x30 = ContentFOTA`)
4. Приёмник собирает чанки, проверяет подпись APK (тот же keystore)
5. Если подпись верна → предложить установку

### Безопасность
- APK подписан тем же ключом что на Play Store
- Проверка signature перед установкой (Android PackageManager)
- Нельзя "откатить" на старую версию (build number только вверх)
- Размер APK ~28МБ → ~280 чанков по 100КБ → передача за ~5 минут через LAN

### Один человек скачал через VPN → вся mesh обновилась
Именно так. Как настоящий blockade runner.

1. Альтернативный набор `--bg`, `--text`, `--surface` и т.д.
